Token 概览

所有输入端 token 包含三个部分：fresh（全新、未缓存）、cache read（从缓存读取）和 cache write（写入缓存供后续使用）。Cache read 和 cache write 都是输入端概念——它们描述的是提示词/上下文的处理方式，而非模型的输出。

不同 Provider 上报的字段有所不同：Anthropic 上报全部三项；OpenAI 不上报 cache write；Codex 仅上报总量。详见下方 Provider 映射。

Token 组成

Provider API 的原始 token 字段。输入端 = fresh + cache read + cache write。

指标 定义 公式 Anthropic OpenAI
Input Fresh 未缓存的新 token，即实际的"新"输入成本。 input_tokens input_tokens input_tokens − cached_tokens
Cache Read 缓存命中的输入 token，比 fresh 更便宜。 cached_input_tokens cache_read_input_tokens cached_tokens（prompt）
Cache Write 写入缓存的输入 token，一次性成本。 cached_output_tokens cache_creation_input_tokens 不上报
输入端合计 所有输入端分类之和。 fresh + R + W 三项之和 求和（无 cache_W）
Output Visible 聊天输出中的 token（不含 reasoning）。 output_tokens output_tokens completion_tokens − reasoning
派生指标

由原始字段计算得出。全部定义在 index/metrics.py 中。除零时返回 —。

指标 公式 解读
Cache Reuse cache_read / (fresh + R + W) 输入中有多少来自缓存。越高越高效。
Cache Write Ratio cache_write / (fresh + R + W) 正在构建缓存的占比。高表示活跃构建缓存中。
Output Ratio output / (fresh + R + W) 输出与输入之比。过低（<5%）可能意味着上下文膨胀。
Tools/Round tool_calls / assistant_msgs 每轮平均工具调用次数。高可能意味着任务较复杂。
Tokens/Round (input_side + output) / assistant_msgs 每轮平均 token 数。高表示上下文窗口较大。
Tokens/Minute (input_side + output) / (duration / 60) Token 吞吐量。低值 + 长耗时可能意味着存在空闲。
Failed/Session failed_tools / sessions 每会话平均失败工具调用次数，不是百分比。

# 限制 影响
1 Token 计数因 Provider 而异 Anthropic 使用 cl100k 编码，OpenAI 使用自己的编码器。跨 Provider 比较应视为近似值。
2 OpenAI/Codex 缺少 cache write 这些 Provider 不上报 cache write 字段。输入端合计不包含此项——不是零，只是未上报。
3 部分解析失败的会话 日志损坏或格式变更可能导致解析不完整。Token 计数可能被低估。
4 持续时间精度 基于日志时间戳推算。可能包含进程挂起、笔记本休眠或轮转间隔导致的空闲时间。
5 Codex token 估算 仅有 tokens_used（总量）。每轮明细为均匀分配——是一种近似。
6 异常阈值为经验值 使用 P90/P95 且最低 20 会话样本。低于此值时使用回退值，可能不适用于所有工作流。
Session Anomalies

检测整段 session 是否需要关注，用于 /dashboard 的 Needs Attention、/sessions 的异常筛选、session 详情顶部 anomaly banner。

类型 规则 严重度
Long Duration 总活跃时间（Model Exec + Tool Exec，合并并行重叠）>= 1h（警告）/ >= 2h（严重） warning / critical
Failed Tools 失败比例 >= 15%（警告）；失败比例 >= 25%（严重）。单次失败不会触发。 warning / critical
Cache Creation Cache creation（cache_creation_input_tokens）>= 200K（info）/ >= 500K（warning）。表示该 session 向 prompt cache 写入了大量上下文，供后续 round 复用。是 info/warning 级别——不是问题指标。在 Hotspots 标签页中可查看每轮的写入分布。 info / warning
Round Signals

检测单个 ROUND 是否值得展开查看，用于 session 详情页 Timeline 表格的 SIGNALS 列。阈值与 session anomaly 不同，更敏感。

Signal 规则 严重度
failed tool 单轮有失败工具调用：1-2 个 warning，>= 3 个 critical warning / critical
llm error 单轮有 LLM error：1-2 个 warning，>= 3 个 critical warning / critical
long tool 单个工具耗时 >= 5 min warning
tool burst 单轮工具调用 >= 20（排除 top 3 工具占 >= 90% 的 tight loop） warning
high write 单轮 cache creation >= 300K tokens（cache_creation_input_tokens，写入 prompt cache 供后续 round 复用） warning
large input 单轮 input >= 200K 且占 session input >= 50% warning
