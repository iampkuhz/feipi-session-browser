# 提案：仓库文档瘦身

## 背景

仓库中存在大量历史改造记录、版本化 UI 设计说明、旧验收手册和 harness 维护说明。它们会增加 agent 上下文、人工审查成本和长期维护面。

## 目标

- 仓库只保留当前代码和当前工作流需要的规约。
- 删除历史版本、迁移过程、旧兼容逻辑和已完成 change 记录。
- 保留并维护 `docs/ui/` 作为页面功能要求集合。
- 将 harness 文档收敛到机器执行入口和少量当前规则。
- 生成 `tmp/` 下的瘦身审查报告，列明修改范围。

## 非目标

- 不修改产品运行逻辑。
- 不重写 CSS/JS 当前实现。
- 不删除测试代码和质量门脚本本身。
