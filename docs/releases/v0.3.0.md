# v0.3.0

本版本聚焦会话浏览体验、Token 术语一致性和发布范围收敛。v0.3.0 暂不发布 wheel，也暂不发布或推荐镜像；发布资产先保留 source archive，后续在安装和镜像路径验证充分后再扩展。

## 使用体验

- Sessions 搜索改为服务端全量查询：输入时先快速过滤当前页，并在短暂防抖后查询全库，避免只能搜索当前分页的 20 条记录。
- Session Detail 的 attribution modal 做了布局打磨，减少弹窗内容拥挤和宽屏阅读负担。
- Token 与 attribution 文案进一步统一，减少旧术语和重复概念，让统计口径更容易理解。

## Token 与归因

- 统一 Token accounting 的命名和展示口径，围绕 Input Fresh、Cache Read、Cache Write、Output 等概念清理历史命名。
- 收敛 attribution taxonomy 与各 agent 的 request/response bucket 处理，减少不同 agent 页面之间的术语差异。
- 清理旧字段兼容和冗余命名，为后续更稳定的跨 agent 对比做准备。

## 质量与维护

- 收紧测试 skip 策略：required、selected、full/release regression 中出现 skipped outcome 不再视为通过。
- 清理历史 skip，并补充 `noTestSkips` gate，防止新增 `pytest.skip`、`pytest.mark.skip`、Playwright `test.skip()` 等绕过验证。
- 去除重复 dev 启动脚本，统一维护入口到 `scripts/session-browser.sh`。
- 优化 agent / OpenSpec / harness 规则，使后续大模型协作更容易限定范围并保留验证证据。

## 文档与发布范围

- README 已收敛为面向使用用户的命令行启动说明，只保留 `deps`、`scan`、`serve`、访问地址、常用环境变量和隐私说明。
- README 不再展示本地 Podman 部署、镜像、发布流程、harness 和内部实现原理；这些内容后续会放到更适合维护者的文档或脚本帮助里。
- v0.3.0 发布流程先暂停 wheel，避免在验证不充分前承诺 wheel 安装路径。
- v0.3.0 暂不处理镜像发布；镜像构建和部署说明后续调试稳定后再单独补齐。

## 发布资产

- GitHub Release：`v0.3.0`
- Source archive：`feipi_session_browser-0.3.0.tar.gz`
- 暂不包含：wheel、GHCR 镜像、latest 镜像标签

## 启动方式

```bash
./scripts/session-browser.sh deps
./scripts/session-browser.sh scan
./scripts/session-browser.sh serve
```
